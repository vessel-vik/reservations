Data Flow

How data moves through the Finance Hub
KPI Strip
→ /api/reports/accounting (existing)
Returns totalIncome, totalExpenses, netProfit, netVat
→ /api/budgets?month=2026-03 (new)
Returns category limits for over-budget detection
Re-fetches on period toggle (Today/Week/Month/Quarter)
Expense Drawer (create)
1. Zod validates locally — shows inline errors
2. If receipt: POST /api/expenses/upload (multipart)
→ Appwrite Storage → returns receiptUrl
3. POST /api/expenses with receiptUrl in body
4. Toast success → close drawer → refetch list
5. If error → toast error → drawer stays open
Budget Meters
→ GET /api/expenses (existing) — actuals per category
→ GET /api/budgets (new) — limits per category
→ budget-utils.compareBudgetToActual()
Returns % used, overage amount, status (ok/warn/over)
Color: green <80% · amber 80–100% · red >100%
Reports Chart
→ GET /api/admin/analytics (existing)
Returns daily revenue breakdown
→ GET /api/expenses grouped by invoiceDate
→ Merged into daily [ date, revenue, expenses, profit ]
CSV export: client-side string build, Blob download
PDF export: window.print() with print stylesheet
