Stock Auto-Decrement Flow (POS → Admin)

What happens when a POS order is confirmed
🧾
1. Order confirmed
Staff taps "Confirm" in POS — createOrder() called with cart items
→
📦
2. Stock decremented
For each cart item: stock -= quantity via Appwrite PATCH. If stock hits 0: isAvailable = false also set
→
⚠️
3. Low-stock strip updates
Admin's Menu & Stock tab refetches items. Low-stock strip shows newly at-risk items. POS hides unavailable items (Sub-project 3)
Decrement is best-effort: if Appwrite fails during stock update, the order is still saved (order creation is not rolled back). A failed decrement is logged to console — full transactional rollback is v2 scope.
